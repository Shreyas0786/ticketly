# Contributing & Feedback

Thanks for using Ticketly. Here's how this project works.

## Code contributions

Ticketly is source-visible but **not** open to code contributions. I'm not
accepting pull requests, so please don't open one — it will be closed with a
pointer back here. (See the [LICENSE](LICENSE) for the usage terms.)

## What's welcome

The best way to help is to tell me what's working and what isn't:

- **Bug reports** — something broke or behaved unexpectedly? Open an issue.
- **Feature requests** — want it to do something it doesn't? I'd like to hear it.
- **Feedback** — rough edges, confusing wording, ideas — all of it helps.

All of that goes through
**[GitHub Issues](https://github.com/Shreyas0786/ticketly/issues)**. There are
short templates for bugs and features to make it quick.

I read every issue. Thanks for taking the time.
